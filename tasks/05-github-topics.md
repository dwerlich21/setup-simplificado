# Tarefa 05 — Topics e description no repositório GitHub

## Problema

O repositório não tem topics (tags) nem uma description atrativa. Isso prejudica a discoverability (quando recrutadores buscam por tecnologias específicas) e a primeira impressão visual na página do repo.

## O que fazer

### 1. Adicionar description do repositório

Ir em **Settings** do repositório e definir:

**Description:**
> Full-stack admin boilerplate — Vue 3 + Laravel 12 with cookie-based auth, RBAC, audit logging, notifications, reports (PDF/Excel), and Kanban views.

**Website:** (se tiver deploy, colocar a URL aqui)

### 2. Adicionar topics

Na página principal do repositório, clicar na engrenagem ao lado de "About" e adicionar os seguintes topics:

```
vue3
laravel
laravel-12
sanctum
pinia
bootstrap
mysql
rbac
audit-log
admin-dashboard
boilerplate
fullstack
rest-api
crud
kanban
php
javascript
vite
```

### Por que isso importa

- Recrutadores buscam no GitHub por tags como `vue3`, `laravel`, `admin-dashboard`
- O repo aparece em listagens de projetos por tecnologia
- Mostra que você se preocupa com a apresentação e organização do projeto

## Onde fazer

Tudo é feito pela interface web do GitHub:
1. Abrir `https://github.com/dwerlich21/setup-simplificado`
2. Clicar na engrenagem ao lado de "About" (sidebar direita)
3. Preencher Description, Website e Topics
4. Salvar

## Estimativa: ~5 minutos
